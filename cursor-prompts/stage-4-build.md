# Промпт этапа 4: Knowledge Base и role-specific prompt system

**Назначение**: Задать контекст и задачи для подключения локальной knowledge base (Adobe Help Center + docsforadobe) и трёх проекций знаний для generator, validator и repair. Использовать вместе с shared-project-context.md и docs/archive/plans/staged-implementation-plan.md (этап 4).

---

## Контекст

- Этап 3 выполнен: multi-pass pipeline работает; пользователь видит один финальный ответ; стадии отображаются в статусной строке.
- Цель этапа 4: один корпус знаний (primary — Adobe Help Center, secondary — docsforadobe); три проекции для ролей generator, validator, repair. На каждом шаге pipeline использовать соответствующую проекцию для сборки контекста (system/context сообщений).

---

## Задачи (кратко)

1. Определить формат и источник корпуса: расширение текущего aeDocsIndex или отдельный индекс; данные из Adobe Help Center и docsforadobe. Документировать процесс сборки корпуса.
2. Реализовать три проекции поверх одного корпуса: для generator (генерация по запросу и контексту), для validator (проверка API и правил), для repair (исправление с опорой на документацию и замечания валидации). Проекции могут быть разными запросами к индексу, разными правилами отбора/ранжирования или разным форматированием одних и тех же сниппетов.
3. Интегрировать проекции в pipeline: на шагах generate, validate-1, validate-2 и repair подставлять соответствующую проекцию в сборку system/context сообщений (аналог текущего BUILD_DOCS_CONTEXT_MESSAGE с ролью). Не менять формат ответа пользователю и ручной Apply.
4. Обеспечить деградацию: при отсутствии данных или индекса панель ведёт себя как без docs (как сейчас при пустом индексе), без падений.
5. Definition of done: generator, validator и repair получают контекст из knowledge base через свои проекции; документация по корпусу и проекциям зафиксирована.

---

## Что здесь не делаем

- Не меняем оркестрацию в main.js (multi-pass уже введён на этапе 3).
- Не меняем Apply и формат финального вывода в чат.
- Не вводим новые модели; только использование существующего корпуса и проекций.
