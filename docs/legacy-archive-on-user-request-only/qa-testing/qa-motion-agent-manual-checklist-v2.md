# Manual Test Checklist — v2 Improvements

Все тесты проводятся в After Effects с открытой панелью AE Motion Agent.
Перед началом: перезагрузить панель (закрыть и открыть заново) чтобы подтянулись все изменения.

---

## 1. Базовый запуск панели

- [ ] Панель загружается без ошибок в консоли CEP (Developer Tools → Console)
- [ ] Status line показывает "Ready"
- [ ] Tooltips видны при наведении на кнопки (New, Rename, Clear, Clear All, Undo, Send, Stop)
- [ ] Placeholder в textarea содержит примеры запросов

---

## 2. Expression Error Detection (#2)

**Тест A — ошибка экспрешена:**
1. Создать композицию с solid layer
2. Отправить агенту: "Примени экспрешен `undefinedFunction()` к Position слоя 1"
3. Ожидание: tool call `apply_expression` возвращает `ok: false` с текстом ошибки
4. Ожидание: агент видит ошибку и сообщает о ней (или пытается исправить)

**Тест B — успешный экспрешен:**
1. Отправить: "Добавь wiggle(3, 25) к Position слоя 1"
2. Ожидание: `apply_expression` → `ok: true`
3. Ожидание: в AE на слое появился wiggle, нет красной иконки ошибки

---

## 3. SourceText Expressions (#1)

1. Создать текстовый слой в AE
2. Отправить: "Напиши экспрешен для Source Text — счётчик от 0 до 100 за 3 секунды"
3. Ожидание: агент применяет expression к Text>Source Text
4. Ожидание: экспрешен работает без ошибки, в AE текст считает от 0 до 100
5. Проверить: нет красной иконки ошибки на Source Text

**Тест B — мультистрочный текст:**
1. Отправить: "Напиши экспрешен для Source Text — две строки: текущий кадр и время"
2. Ожидание: агент использует `\r` для переноса строки, экспрешен работает

---

## 4. get_expression Tool (#18)

1. Вручную добавить экспрешен `wiggle(2, 50)` на Position любого слоя
2. Отправить агенту: "Прочитай текущий экспрешен на Position слоя 1"
3. Ожидание: агент вызывает `get_expression`, в результате видно `expression: "wiggle(2, 50)"`, `expressionEnabled: true`

**Тест B — ошибочный экспрешен:**
1. Вручную написать сломанный экспрешен на Position: `blabla()`
2. Отправить: "Что за экспрешен на Position слоя 1?"
3. Ожидание: `get_expression` возвращает `expressionError` с текстом ошибки AE

---

## 5. Comp Summary — 3D, dimensions, compact, filters (#4, #19, #20)

**Тест A — 3D и размеры:**
1. Создать компу с solid layer (500x300) + text layer + включить 3D на одном из них
2. Отправить: "Опиши текущую композицию"
3. Ожидание: в результате `get_detailed_comp_summary` видно поля `threeDLayer`, `width`, `height`

**Тест B — compact mode (большой комп):**
1. Создать или открыть компу с 10+ слоями
2. Отправить: "Покажи список всех слоёв кратко"
3. Ожидание: агент вызывает `get_detailed_comp_summary` с `compact: true`
4. Ожидание: результат содержит минимальные данные (index, id, name, type)

**Тест C — фильтр по типу:**
1. Отправить: "Покажи только текстовые слои"
2. Ожидание: вызов с `layer_type: "text"`, возвращаются только текстовые слои

## 6. Tool Call History (#15)

1. Отправить: "Создай текстовый слой с надписью Hello"
2. Дождаться выполнения
3. Отправить второй запрос: "Измени текст этого слоя на World"
4. Ожидание: агент помнит, что создал текстовый слой, и знает его index/id
5. Ожидание: агент НЕ вызывает get_detailed_comp_summary заново (или вызывает, но знает контекст)

## 7. Cancel Button (#13)

1. Отправить сложный запрос: "Создай 10 слоёв с разными цветами и анимируй каждый"
2. Во время выполнения (пока видны tool calls) нажать **Stop**
3. Ожидание: агент останавливается, в чате сообщение "[Agent cancelled by user.]"
4. Ожидание: кнопка Stop скрывается, Send активна

---

## 8. Batch Undo (#26)

1. Отправить: "Создай 3 solid слоя разных цветов и назови их Red, Green, Blue"
2. Дождаться выполнения (3+ tool calls: create_layer × 3 + rename × 3)
3. Нажать кнопку **Undo** в панели
4. Ожидание: статус показывает "Undo: N actions reverted" (где N — количество мутирующих tool calls)
5. Ожидание: ВСЕ созданные слои удаляются
6. Примечание: AE не поддерживает единый undo group через несколько evalScript вызовов, поэтому панель считает мутирующие tool calls и выполняет N × Cmd+Z за один раз

4. Ожидание: thinking indicator показывает "Agent [3]: calling add_keyframes..."

## 13. Conversation Pruning (#14)

2. Отправить ещё один запрос
3. Ожидание: запрос проходит без ошибки "context length exceeded"
4. Ожидание: агент всё ещё понимает контекст (хотя бы последние сообщения)

## 14. Host Script Loading (#16)

1. Открыть CEP Developer Tools (Console)
2. Отправить первый запрос
3. Отправить второй запрос
4. Ожидание: в консоли нет повторной загрузки/парсинга host script
5. Ожидание: второй запрос выполняется быстрее первого

## 15. Expression Retry (#9)

1. Отправить заведомо сложный запрос: "Напиши экспрешен для Position — пружинящее движение с overshoot на основе кейфреймов"
2. Если первая попытка агента с ошибкой:
3. Ожидание: агент читает текст ошибки из `apply_expression` результата
4. Ожидание: агент делает вторую попытку с исправленным экспрешеном
